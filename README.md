# 1.Use Case Diagram
A Use Case Diagram shows how users (actors) interact with a system and what functions or services the system provides.
It focuses on system requirements from the user's perspective.

![1](https://github.com/user-attachments/assets/bac139cb-1fe7-4597-8f85-fad7c58d0120)

# 2. Class Diagram
A Class Diagram represents the structure of a system by showing classes, their attributes, methods, and relationships between classes.
It is mainly used in object-oriented design.

![2](https://github.com/user-attachments/assets/ffe72e25-a24e-4752-88a3-e290a48915c7)

# 3. Sequence Diagram
A Sequence Diagram shows how objects interact with each other in a specific order over time.
It focuses on the sequence of messages exchanged between objects to complete a process.

![3](https://github.com/user-attachments/assets/7a3b5eb0-d4eb-45d0-b8ee-8335da043a9c)

# 4. Activity Diagram
An Activity Diagram illustrates the workflow or step-by-step flow of activities in a process.
It is similar to a flowchart and shows decisions, loops, and parallel processes.

![4](https://github.com/user-attachments/assets/5237974c-e355-4789-b1c8-ce9d0e62a221)

# 5. State Diagram (State Machine Diagram)
A State Diagram shows the different states of an object and how it changes from one state to another based on events.
